Marque cada termo: J = jargão (entra na blocklist, bloqueia em pergunta

Critério: J = vocabulário de sistema que o cliente estranharia
(subpilar, ratio, Lastro, Capital Relacional). C = palavra natural que o
método usa mas o cliente entende (precisão, disponibilidade, atendimento).

Régua conservadora aplicada: na dúvida, C. Falso-bloqueio (barrar pergunta
boa) frustra o usuário; deixar passar um termo limítrofe não causa dano.
Termos que são palavra comum E rótulo de método (crítico, estável,
consistência, severidade, prioridade...) ficam C — o jargão real desses em
pergunta a cliente é raro, e o risco de falso-bloqueio supera o ganho.

Dois pontos para o Code resolver antes de aplicar


Termos compostos / código vs nome. Vários J são compostos
("P1 — Calibração da Promessa", "Origem (filtro do Plano de Ação)"). O Code
deve bloquear o termo-RAIZ que é jargão (o código "P1", "N5"; "agrupamento")
e NÃO o nome comum embutido ("calibração", "acessibilidade", "empatia").
Bloquear o composto inteiro causaria falso-bloqueio de palavras comuns.
Limítrofes deixados C por segurança (Reclassificação, Leitura
diagnóstica): se o Alexandre preferir mais rígido, viram J. Decisão dele.
